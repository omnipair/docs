# Code Reference

Welcome to the Omnipair Solana program code reference. This documentation provides detailed information about the Anchor program implementation, including instructions, accounts, errors, and constants.

## Program Overview

The Omnipair program is a Solana Anchor program that implements a Generalized Automated Market Maker (GAMM) with integrated lending functionality. It enables:

- **Spot Trading**: Constant-product AMM swaps
- **Liquidity Provision**: Add and remove liquidity from pools
- **Margin Lending**: Borrow against collateral with dynamic interest rates
- **Flash Loans**: Uncollateralized loans that must be repaid in the same transaction
- **Liquidations**: Automated liquidation of undercollateralized positions

## Architecture

The program is organized into several modules:

- **Instructions**: Entry points for program interactions
  - Liquidity management (initialize, add, remove)
  - Spot trading (swap)
  - Lending operations (collateral, borrow, repay, liquidate, flashloan)
  - Futarchy governance (authority management, fee distribution)
  - View functions (read-only data access)
- **State**: On-chain account structures
  - `Pair`: Pool state and reserves
  - `UserPosition`: User collateral and debt tracking
  - `FutarchyAuthority`: Protocol governance and fee configuration
  - `RateModel`: Interest rate model configuration
- **Utils**: Helper functions for math, token operations, and account management
- **Constants**: Protocol-wide constants and configuration

## Key Features

### Oracle-less Pricing
- Uses a **double-EMA system** (symmetric + directional) for debt valuation
- Pessimistic pricing: `min(directional_ema, symmetric_ema)`
- No external oracle dependencies

### Impact-Aware Collateral Factors
- Dynamic collateral factors derived from AMM reserves with price impact simulation
- 5% LTV buffer between borrow limit and liquidation threshold
- `applied_min_cf_bps` protection for existing positions

### Utilization-based Interest Rates
- Interest rates adjust based on pool utilization
- Fully parameterizable exponential rate model with min/max caps
- Target utilization bands with configurable bounds

### Isolated Collateral Pools
- Each pair operates as an isolated lending market
- No cross-pool risk exposure
- Per-pair reduce-only mode available

### Solvency Invariant
- Three-tier reserve accounting: virtual, cash, and debt
- State transition equation: `ΔR_virtual = ΔR_cash + ΔR_debt`
- Protocol fees tracked as `vault_balance - cash_reserve`
